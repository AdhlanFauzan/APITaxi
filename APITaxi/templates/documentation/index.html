Vue d'ensemble

Interactions moteur ⇄ operateur ⇄ taxi

L'objectif du Registre Géolocalisé des Taxis (RGT) est de mettre en relation les clients des moteurs de recherche et des taxis outillés par des opérateurs de taxis. Pour ce faire, le RGT sert d'intermédiaire entre les moteurs de recherche et les opérateurs de taxis.
Afin de garantir une fiabilité maximale, la communication de la position et de la disponibilité des taxis doit être faite directement depuis un équipement embarqué dans le taxi vers le RGT. Toutes les autres interactions avec le RGT peuvent être faites depuis les infrastructures centrales des opérateurs de taxis (resp. des moteurs de recherche).

Mise en oeuvre de l'API

Toutes les API présententées ici, à l'exception de celle permettant la remontée de la géolocalisation du taxi, sont des API web REST sur HTTPS.
Tous les retours sont encodés en JSON.
